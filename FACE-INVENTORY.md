## Repo facts (verified)
- **All 18 are truly resource-only** — every manifest has `android:hasCode="false"`, zero `<activity>/<service>/<receiver>/<provider>`, `com.google.android.wearable.standalone=true`, and the `com.google.wear.watchface.format.version` property. **This already satisfies WFP's no-code / single-face rule.** The ONLY change per face to make it WFP-pushable is the `applicationId`.
- Each face is an independent top-level Gradle project (own `settings.gradle`/`build.gradle` + `app/` module). Face WFF doc at `<Face>/app/src/main/res/raw/watchface.xml`; preview at `res/drawable[-nodpi]/preview.png`.
- **Signing today:** two committed throwaway DEBUG keystores (`arclight-debug.keystore` for the 2 Arclight faces, `fable-debug.keystore` for the other 16), well-known creds (`android`/`androiddebugkey`/`android`). Explicitly NOT Play keys — repo comments note "Play signing will use a separate keystore provisioned via secrets."
- **CI today** (`.github/workflows/build.yml`): JDK17 + Gradle 8.9; validates each `watchface.xml` with Google's `wff-validator.jar`; builds `assembleDebug` (so releases are debug-signed today); per-product tag `<slug>-v*` → GitHub Release. No secrets, no AAB/Play step.
- Face projects are generated by `tools/make-projects.mjs`; assets by `tools/assets/`.

## Implication for Dialed (Phase 0)
Repackaging is trivial: take each face's `res/**` + manifest (WFF content unchanged), rebuild with `applicationId = com.dialed.app.watchfacepush.<series>.<face>` signed with a **dedicated shared faces keystore** (must differ from the `:app` key), run the WFP validator CLI (`--package_name=com.dialed.app`) to mint the token, then bundle APK + token into `:app/assets`. **Tokens require a real APK build → this happens in CI (no local Android toolchain on this machine).**
